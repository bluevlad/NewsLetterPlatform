ADMIN GUIDE

뉴스레터 발송 이력 가이드

전체 테넌트의 뉴스레터 발송 결과를 추적하고 분석합니다

1. 서비스 소개

발송 이력 페이지는 뉴스레터 발송의 성공/실패 여부, 수신자 정보, 오류 내용 등을 상세히 기록하고 조회할 수 있는 관리자 기능입니다. 통합 이력 페이지에서는 전체 테넌트의 발송 결과를 하나의 화면에서 모니터링할 수 있습니다.

대상 사용자

서비스 관리자 운영 담당자

2. 주요 기능

📊

7일 요약 대시보드

최근 7일간 전체 발송 건수, 성공/실패 수, 성공률을 일별 테이블로 한눈에 파악합니다.

🔍

다중 필터링

테넌트, 날짜 범위, 성공/실패 상태를 조합하여 원하는 이력만 정확히 조회합니다.

📋

상세 이력 테이블

발송 시간, 테넌트, 수신자 이메일, 제목, 상태, 오류 메시지를 상세히 확인합니다.

🔄

페이지네이션

대량의 발송 이력을 20건 단위로 페이지 분할하여 빠르게 탐색합니다.

3. 사용 방법

  1. 발송 이력 페이지 접속
    관리자 로그인 후 사이드바에서 All History를 클릭하거나, /admin/history로 직접 접속합니다. 특정 테넌트의 이력만 보려면 /admin/{tenant}/history로 접속합니다.
  2. 7일 요약 확인
    페이지 상단의 Daily Summary 테이블에서 최근 7일간 전체 발송 현황을 확인합니다.
  3. 필터 조건 설정
    상태 필터(All / Success / Failed), 테넌트 선택, 시작일~종료일을 설정한 후 Filter 버튼을 클릭합니다.
  4. 이력 상세 확인
    테이블에서 각 발송 건의 시간, 수신자, 제목, 성공/실패 상태, 오류 메시지를 확인합니다.
  5. 페이지 이동
    하단 페이지네이션으로 이전/다음 페이지를 탐색합니다.

5. 주의사항

참고

발송 이력은 관리자 인증이 필요한 페이지입니다. 미인증 상태에서 접근하면 로그인 페이지로 자동 이동합니다.

참고

테넌트별 이력 페이지(/admin/{tenant}/history)와 통합 이력 페이지(/admin/history)는 동일한 데이터를 조회합니다. 통합 페이지에서 테넌트 필터를 적용하면 테넌트별 페이지와 동일한 결과를 볼 수 있습니다.

주의

발송 이력은 삭제할 수 없으며, 감사(audit) 목적으로 영구 보관됩니다.

6. 자주 묻는 질문

Q. FAIL 상태의 발송 건은 자동으로 재시도되나요?
아닙니다. 실패한 건은 자동 재시도되지 않습니다. 수동 발송 페이지(/admin/{tenant}/send)에서 다시 발송할 수 있습니다.
Q. 특정 기간의 이력만 보려면?
Date From과 Date To에 날짜를 입력하고 Filter 버튼을 클릭하면 해당 기간의 이력만 표시됩니다.
Q. 오류 메시지가 잘려서 보입니다
Error 컬럼은 공간 제약으로 긴 메시지가 말줄임 처리됩니다. 전체 오류 내용은 마우스를 올려 툴팁으로 확인하거나 서버 로그를 참조하세요.
Q. Daily Summary는 어떤 기준인가요?
최근 7일간 UTC 기준 일별 발송 건수를 집계합니다. Total은 전체 발송 수, Success는 성공 수, Failed는 실패 수, Rate는 성공률입니다.
Q. 이력 데이터를 CSV로 내보낼 수 있나요?
현재 발송 이력의 CSV 내보내기 기능은 지원하지 않습니다. 구독자 목록은 구독자 관리 페이지에서 CSV 내보내기가 가능합니다.
